## Critical lessons

### chrome.storage in web/dev mode
- `chrome.storage.local` exists in ANY Chrome tab — calls hang forever
- Must check `chrome.runtime?.id` (only non-null in real extensions)
- Solution: `src/store/context.ts` exports `IS_EXTENSION` boolean

### createEffect in async functions
- NEVER call `createEffect` inside an async function like `initConnection`
- Each call creates a new effect that never cleans up
- Multiple calls = multiple competing effects writing to same signals
- Effects belong at component top level

### Automerge ImmutableString
- Automerge 3.x stores strings as `ImmutableString` objects, not plain JS strings
- `typeof value === 'string'` returns `false`
- Always `String(value)` before passing to bcryptjs or serialising

### Dev server vs built app
- Vite HMR does NOT reliably reload non-component modules (auth.ts, storage.ts)
- Always build (`npm run build`) and serve static (`python3 -m http.server`) for testing
- Even then, browser module cache requires incognito window for clean test
