## Sandbox/Pi permissions fix — applied

Root cause: sandbox and Pi share the same filesystem. Sandbox agent runs as a different
user; created files conflict with pi user ownership.

Three fixes applied:

1. **`pnpm build` clears dist first:** `{ sudo rm -rf dist 2>/dev/null || rm -rf dist 2>/dev/null || true; }` before vite build. Axle's pattern from idea#17.
2. **Vite cache in /tmp:** `cacheDir: '/tmp/idea-console-vite-cache'` in vite.config.ts. Moves dep-optimisation cache out of node_modules — always writable by all users.
3. **CI=true pnpm install:** When I (sandbox agent) run pnpm install, always prefix with `CI=true pnpm install`. Prevents pnpm aborting in non-TTY environments. ⚠️ This is MY discipline — not in package.json.
